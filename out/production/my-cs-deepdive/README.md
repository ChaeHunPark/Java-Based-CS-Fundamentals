## 💡 핵심 용어 정리

| 용어 | 설명 | 비유 |
| :--- | :--- | :--- |
| **Encapsulation** | 데이터와 기능을 하나로 묶고 외부 접근을 제한함 | 주방 내부 조리 과정을 손님에게 감추기 |
| **Singleton** | 클래스의 인스턴스가 딱 하나만 생성되도록 보장 | 주방에 단 하나뿐인 메인 셰프 |
| **Eager Init** | 클래스 로딩 시점에 인스턴스를 미리 생성하는 방식 | 오픈 전 미리 요리를 준비해두기 |
| **Thread-Safe** | 멀티스레드 환경에서도 데이터가 안전하게 유지됨 | 여러 요리사가 동시에 들어와도 안 꼬임 |
| **Static Final** | 상수로 선언하여 변경 불가능한 유일한 참조를 만듦 | 주방의 붙박이 냉장고 위치 고정 |
| **synchronized** | 메서드나 블록에 자물쇠를 채워 동시 접근을 막음 | 화장실 칸의 잠금 장치 |
| **Interface** | 객체가 수행해야 할 기능(규격)을 정의한 명세서 | 요리사가 지켜야 할 표준 레시피 |
| **Factory** | 객체 생성의 복잡한 과정을 대신 처리해 주는 클래스 | 주문을 받으면 요리를 내주는 주방 |
| **OCP** | 개방-폐쇄 원칙 (확장에는 열려 있고 수정에는 닫힘) | 기존 메뉴판을 안 바꾸고 새 메뉴 전용 주방을 증축하는 법 |
| **Abstraction** | 세부 구현을 감추고 공통 규격만 드러냄 | "어떻게 만드는지" 몰라도 "만들어진다"는 사실만 믿고 주문하기 |
| **Decoupling** | 구성 요소 간의 의존성을 줄여 독립성을 높임 | 메뉴가 바뀌어도 홀 서빙 방식은 그대로인 상태 |
| **Strategy** | 객체의 행위를 클래스로 캡슐화한 것 | 게임 캐릭터의 교체 가능한 무기 |
| **Context** | 전략을 사용하는 주체 (전략을 실행하는 환경) | 무기를 사용하는 게임 캐릭터 |
| **Composition** | 필드로 객체를 보유하여 기능을 빌려 쓰는 방식 | 캐릭터가 무기를 '가지는' 관계 |
