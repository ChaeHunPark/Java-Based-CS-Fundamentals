# 🌐 Network 탐구 노드
> **"소켓부터 HTTP까지, 데이터의 흐름을 코드로 증명한다"**

## 📝 학습 리스트

### **Step 01: OSI 7계층 & TCP 소켓 통신** [[코드]](./Step01_Socket.java)
- **내용**: Java `ServerSocket`을 이용한 1:1 TCP 통신 구현 및 전송 계층 이해.
- **핵심 통찰**:
    1. **TCP 3-Way Handshake**: `new Socket()`과 `accept()` 사이, OS 레벨에서 발생하는 연결 확립 절차 이해.
    2. **Handshake의 주체**: 이 과정은 애플리케이션(Java)이 아닌 운영체제(OS)의 TCP 스택에서 자동으로 수행됨.

### **Step 02: HTTP Request Header 분석** [[코드]](./Step02_HttpServer.java)
- **내용**: TCP 소켓 위에 HTTP 규격을 얹어 웹 브라우저(Chrome)와 직접 통신.
- **핵심 통찰**:
    1. **HTTP는 텍스트다**: 모든 웹 통신은 결국 정해진 규격(Method, Header, Body)을 따르는 문자열의 전송임.
    2. **다중 요청과 파비콘**: 브라우저는 메인 페이지 이외에도 아이콘(`favicon.ico`) 등을 가져오기 위해 별도의 TCP 연결을 시도함.
    3. **Stateless (무상태성)**: 서버는 각 요청을 독립적으로 처리하며, 이전 요청의 상태를 기억하지 않음.

### **Step 03: Query Parameter 파싱 & 동적 응답** [[코드]](./Step03_QueryParamServer.java)
- **내용**: URL 파라미터(`?key=value`)를 직접 파싱하여 맞춤형 응답 생성.
- **핵심 통찰 (Spring MVC의 실체)**:
    1. **문자열 파싱의 실체**: `split` 등을 사용하여 데이터를 추출하는 과정이 Spring `@RequestParam`의 근본 원리임을 파악함.
    2. **ArgumentResolver**: 요청 데이터를 메서드 인자로 변환해주는 '해결사' 설계의 필요성 인지.
    3. **Dynamic Content**: 고정된 페이지가 아닌, 전달받은 파라미터에 따라 실시간으로 변하는 응답(HTML) 생성.

## 💡 네트워크 & 백엔드 핵심 용어

| 분류 | 용어 | 설명 | 비유 |
| :--- | :--- | :--- | :--- |
| **연결** | **Handshake** | 데이터를 주고받기 전 연결을 확립하는 과정 | "저기요?", "네?", "말씀하시죠" |
| **연결** | **Keep-Alive** | 한 번 맺은 TCP 연결을 끊지 않고 재사용 | 볼일 끝날 때마다 전화 끊지 않고 계속 들고 있기 |
| **특성** | **Stateless** | 서버가 클라이언트의 상태를 보존하지 않는 성질 | 매번 누군지 다시 말해야 하는 키오스크 |
| **처리** | **Parsing** | 가공되지 않은 문자열을 의미 있는 데이터로 추출 | 엉킨 실타래에서 필요한 색깔의 실만 뽑아내기 |
| **처리** | **Dynamic Content** | 사용자 요청에 따라 실시간으로 변하는 응답 | 로그인한 사람의 이름을 띄워주는 환영 문구 |
| **설계** | **ArgumentResolver** | 요청 데이터를 메서드 인자로 변환해주는 해결사 | 외국인의 말을 한국어로 번역해주는 통역사 |
